3 approches d'architecture

Clique sur celle qui te parle le plus — on peut en discuter en terminal

A

JAMstack moderne — ma recommandation ⭐

Next.js (React) + Supabase (BDD PostgreSQL, auth, stockage docs)

Visualisation arbre : React Flow · Vue carte : Leaflet · Timeline : composant custom

Avantages

  • Stack moderne, rapide à builder
  • Supabase gère auth + permissions + stockage docs en un
  • Collaboratif en temps réel natif (Supabase Realtime)
  • Coût faible au démarrage

Limites

  • Dépendance Supabase (SaaS)
  • React Flow a une courbe d'apprentissage pour les arbres complexes
B

Full-stack maîtrisé

Next.js + Node.js API + PostgreSQL auto-hébergé + S3/MinIO pour les docs

Visualisation : D3.js (puissant, sur-mesure)

Avantages

  • Contrôle total sur les données (RGPD, hébergement)
  • D3.js = visualisations très personnalisées
  • Pas de dépendance SaaS

Limites

  • Plus long à mettre en place
  • Infra à gérer soi-même
  • D3.js plus complexe que React Flow
C

GEDCOM-compatible

Next.js + Supabase + support import/export GEDCOM 7 (standard généalogique universel)

Permet d'importer depuis Ancestry, MyHeritage, FamilySearch

Avantages

  • Interopérabilité avec tous les outils généalogiques
  • Les généalogues professionnels adorent GEDCOM
  • Import de données existantes facilité

Limites

  • GEDCOM 7 est complexe à implémenter proprement
  • Scope plus large = délai plus long
  • Sur-engineering si usage 100% interne
